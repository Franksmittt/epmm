# GOD-TIER NEXT.JS 15 ENTERPRISE SPECIFICATION
# Target: High-Ticket Service Based Companies (Law, Med, Finance)

## PILLAR 1: EXTREME SEO ARCHITECTURE
1. Rendering Strategy:
   - Use Incremental Static Regeneration (ISR) for infinite scale.
   - Separate Server Components (RSC) vs Client Components strictly for 0.00 CLS.
2. Programmatic SEO (pSEO):
   - Use Dynamic Routes (`[slug]`) to generate 1,000+ landing pages (e.g., "Service in City").
   - Map "Entities" (Knowledge Graph) instead of just keywords.
3. Edge Optimization:
   - Use Vercel Edge Middleware for geo-personalization and header rewriting (A/B testing) with zero latency.
   - "Invisibility": Logic happens at the edge, not on the client.
4. AI Optimization (LLM Ready):
   - Optimize "Data Density" for AI agents (ChatGPT/Perplexity).
   - Structure HTML so LLMs cite the site as a primary source.

## PILLAR 2: HIGH-CONVERSION UX/UI
1. Perceived Performance:
   - Implement "Optimistic UI" (useOptimistic hook) for instant feedback before server confirmation.
   - Use Skeleton Streaming (Suspense) for purposeful content reveals.
2. Physics-Based Motion:
   - Use "Spring Physics" (Framer Motion) instead of linear CSS transitions.
   - Implement "Layout Projection" for shared element transitions between routes.
3. Conversion Geography:
   - Optimize "Thumb Zone" for mobile CTAs.
   - Use Intercepting Routes for login/modals (keep user in the flow).

## PILLAR 3: TELEMETRY & CONVERSION ENGINEERING
1. Server-Side Tracking:
   - Bypass ad-blockers using Server-Side GTM and HttpOnly Cookies.
   - Implement "Lead Recovery": Capture partial form data before submission using webhooks.
2. Dynamic Personalization:
   - Use Middleware to rewrite content based on UTM parameters (e.g., "Dental Ads" -> Show Implants Hero).
3. Queue Pattern:
   - Offload CRM injections (HubSpot/Salesforce) to background workers (Inngest) to prevent UI blocking.

## PILLAR 4: INDEXATION & GOOGLE SEARCH CONSOLE (GSC)
1. Indexing Velocity:
   - Use the "Google Indexing API" for immediate crawling of high-priority service pages.
   - Implement a "Split Sitemap" strategy for 50k+ URLs.
2. Forensic GSC:
   - Use Regex filters to identify "Striking Distance" keywords (Pos 4-10) and Cannibalization.
   - Monitor Server Logs vs. Googlebot hits to save crawl budget.

## PILLAR 5: METADATA & SCHEMA
1. Dynamic Metadata:
   - Use `generateMetadata` and `opengraph-image.tsx` to auto-generate social images with text overlays.
2. Nested JSON-LD:
   - Inject a "Graph" schema: Organization -> Service -> Review -> Breadcrumb.
   - Fully typed (TypeScript) schema generation.

## PILLAR 6: EXTREME PERFORMANCE (MOBILE 100/100)
1. LCP & Fetch Priority:
   - Use `priority={true}` and `fetchPriority="high"` on Hero images.
   - Inline Critical CSS automatically.
2. CLS & Fonts:
   - Use `next/font` with `adjustFontFallback` to match system fonts perfectly while loading.
3. INP (Interaction to Next Paint):
   - Offload heavy third-party scripts (GTM, Chat) to Web Workers using Partytown.
   - Use the "Facade Pattern" for heavy embeds (YouTube/Maps).

## PILLAR 7: SECURITY & COMPLIANCE
1. Middleware Defense:
   - Implement strict CSP (Content Security Policy) with Nonces.
   - Add Security Headers: HSTS, X-Frame-Options, X-Content-Type-Options.
2. Input Validation:
   - Use Zod schemas for ALL API inputs (Firewall approach).
   - Sanitize all rich-text inputs.
3. Bot Protection:
   - Implement invisible Rate Limiting (Upstash/Redis) and Honey-pot fields.

## PILLAR 8: ACCESSIBILITY (THE LEGAL SHIELD)
1. Headless UI:
   - Use Radix UI or React Aria for accessible primitives (Modals, Dropdowns) that handle focus trapping automatically.
2. Focus Management:
   - Implement "Skip to Content" links.
   - Ensure Focus moves to H1 on route changes.
3. Automated Auditing:
   - Integrate `axe-core` into CI/CD to block deployments if accessibility fails.

## PILLAR 9: THE WHITE-GLOVE CMS
1. Visual Editing:
   - Enable Live Visual Previews (Sanity Presentation Mode) for "Click-to-Edit".
2. Page Builder:
   - Use a "Polymorphic Array" of components (Hero, Pricing, FAQ) for drag-and-drop page building.
3. Safety:
   - Use Singletons for critical pages to prevent layout breakage.
   - Implement "Token-Based Preview" for safe drafting.
